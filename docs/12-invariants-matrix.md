# Матриця інваріантів та цілісності (Invariants Matrix)

1. **Бізнес-ліміт місткості (Twist A - Capacity = 3):**
    * Кількість посилок у статусі `ready_for_pickup` для конкретного `delivery_point_id` не може перевищувати значення поля `capacity` в таблиці `delivery_points`.

2. **Правила часу (Time Rules):**
    * Час у полях `arrived_at` (parcels) та `changed_at` (parcel_status_history) зберігається виключно в **UTC** (ISO 8601).
    * Термін зберігання (Timeslot) не зберігається фізично, а завжди обчислюється як `arrived_at + 3 дні`.
    * Значення `changed_at` для нового статусу завжди має бути більшим або дорівнювати часу попереднього статусу.

3. **Системні інваріанти (System Constraints):**
    * Ідемпотентність: Поле `message_id` в таблиці `processed_logs` гарантує, що жодне повідомлення з черги не буде оброблено двічі (Unique PK).
