# Чеклист доступності (WCAG 2.2 AA) та NFR

## Клавіатурна навігація та фокус
- [x] Повний шлях навігації (створення посилки, прийом, перегляд списку) можна пройти виключно з клавіатури (Tab / Shift+Tab / Enter / Space).
- [x] Логічний порядок табуляції (зверху-вниз, зліва-направо).
- [x] Кожен інтерактивний елемент (посилання, кнопки, інпути) має чітко видимий `focus-ring` (наприклад, outline `2px solid #005fcc`).
- [x] На сторінці `01-home` картки відділень фокусуються цілком як єдиний інтерактивний елемент.

## Семантика та ARIA
- [x] Кожна сторінка має один унікальний заголовок `<h1>`.
- [x] Ієрархія заголовків послідовна (від `h1` до `h3`, без пропусків рівнів).
- [x] Усі поля вводу мають пов'язаний `<label>` (за допомогою атрибута `for` або вкладеності). Плейсхолдер не використовується замість лейбла.
- [x] Модальне вікно (Скасування посилки) містить фокус-трап (focus-trap) — при відкритті фокус не виходить за межі вікна, закривається клавішею `Esc`, а після закриття фокус повертається на кнопку-тригер.

## Робота з формами та помилками
- [x] **Error Summary:** При помилці 422 (напр., невірна вага) фокус програмно переміщується у блок Error Summary над формою.
- [x] Повідомлення про помилки під полями зв'язані з відповідними `input` через атрибут `aria-describedby`.
- [x] При зміні статусу (POST `/api/Parcels/accept`), якщо сервер повертає `409 Conflict` (перевищення Capacity=3), екранний диктор озвучує Global Alert за допомогою `role="alert"` або `aria-live="assertive"`.
- [x] Після натискання "Створити" / "Прийняти" кнопка блокується (запобігання подвійному кліку), але не втрачає фокус несподівано.

## Twist B: Performance-first
- [x] **Skeleton-стани:** Замість зависання екрану під час завантаження деталей відділення (`GET /api/DeliveryPoint/{id}`) відображається skeleton-UI (сірі блоки, що імітують контент).
- [x] **Lazy-render:** Список посилок (`ready_for_pickup`) на сторінці відділення підвантажується порціями (або має пагінацію), щоб уникнути блокування основного потоку браузера.
- [x] **Текст стану:** Під час обробки форми кнопка сабміту отримує текст "Обробка... Не перезавантажуйте сторінку" (або це дублюється у `aria-live="polite"`).
