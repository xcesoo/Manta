# Матриця трасування (Traceability Matrix)

Цей документ демонструє зв'язок між бізнес-вимогами (User Stories), архітектурним дизайном (процеси та сховища DFD), правилами бази даних (Інваріантами) та конкретною реалізацією в коді (API ендпоінти, CQRS-команди/запити та обробка помилок).

| User Story | Design (DFD / ERD) | API Path / Команда | Інваріанти (DB)                                                                                                         | Тести / Error Codes |
| :--- | :--- | :--- |:------------------------------------------------------------------------------------------------------------------------| :--- |
| **US-1** (Прийняття/бронювання комірки)<br>*Фіксація прибуття посилки на відділення та перевірка місткості (capacity).* | **DFD:** Процес **P3** (`accept_parcel_at_point`).<br>**Сховища:** **D1** (parcels), **D2** (delivery_points), **D3** (outbox_messages), **D4** (processed_logs), **D5** (vehicles).<br>**ERD:** Таблиці `parcels`, `delivery_points`, `processed_logs`. | `POST /api/parcels/accept` <br><br> `AcceptParcelAtDeliveryPointCommand` | **Capacity Limit:** Сумарна кількість посилок на точці у статусі `ready_for_pickup` не перевищує ліміт `capacity` (=3). | 202 Accepted (повертає `parcel_guid`) <br> 409 `slot_unavailable` (овербукінг) <br> 409 `location_mismatch` |
| **US-2** (Перевірка слоту `storage`)<br>*Перегляд точного часу закінчення зберігання посилки.* | **DFD:** Читання з **D1** (parcels).<br>**ERD:** Таблиця `parcels` (обчислюване поле `storage` на базі `arrived_at`). | `GET /api/parcels/{id}` <br><br> `GetParcelByIdQuery` | **Time Rule:** Timeslot обчислюється як `arrived_at + 3 дні`. Усі дати в UTC.                                           | 200 OK (повертає DTO посилки) <br> 404 `not_found` |
| **US-3** (Скасування посилки)<br>*Миттєве вивільнення зайнятого місця на відділенні.* | **DFD:** Асинхронне оновлення **D1** (parcels) через запис у **D3** (outbox_messages).<br>**ERD:** Таблиці `parcels`, `outbox_messages`. | `PATCH /api/parcels/{id}/cancel` <br><br> `CancelParcelCommand` *(якщо застосовно)* | **Еволюція статусів:** Перехід дозволено лише з валідних станів. Історія статусів - append-only.                        | 200 OK <br> 409 `wrong_parcel_status` (якщо вже видано `Delivered`) |
| **US-4** (Створення відділення)<br>*Налаштування нової точки з певним лімітом `capacity`.* | **DFD:** Оновлення **D2** (delivery_points).<br>**ERD:** Таблиця `delivery_points`. | `POST /api/deliverypoint` <br><br> `CreateDeliveryPointCommand` | **Constraints:** Обов'язкове встановлення ліміту `capacity` для нової точки.                                            | 201 Created |
| **US-5** (Створення посилки відправником)<br>*Реєстрація нової посилки в системі.* | **DFD:** Процес **P2** (`create_parcel`).<br>**Сховища:** **D1** (parcels), **D3** (outbox_messages).<br>**ERD:** Таблиці `parcels`, `outbox_messages`. | `POST /api/parcels` <br><br> `CreateParcelCommand` | **PII Masking:** Персональні дані клієнта (email, телефон, ім'я) маскуються в логах.                                    | 202 Accepted (повертає `parcel_guid`) |
