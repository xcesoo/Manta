# Матриця трасування (Traceability Matrix)

Цей документ демонструє зв'язок між бізнес-вимогами (User Stories), архітектурним дизайном (процеси та сховища DFD), правилами бази даних (Інваріантами), конкретною реалізацією в коді (API ендпоінти) та інтерфейсом користувача (UX/UI).

| User Story | Design (DFD / ERD) | API Path / Команда | Інваріанти (DB) | Тести / Error Codes | UX Flow | Wireframes (Low-fi) |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **US-1** (Прийняття/бронювання)<br>*Фіксація прибуття посилки на відділення та перевірка місткості (capacity).* | **DFD:** Процес **P3**.<br>**ERD:** `parcels`, `delivery_points`, `processed_logs`. | `POST /api/Parcels/accept` <br><br> `AcceptParcelAtDeliveryPointCommand` | **Capacity Limit:** Сумарна кількість посилок на точці у статусі `ready_for_pickup` не перевищує ліміт `capacity` (=3). | 202 Accepted<br>409 `slot_unavailable`<br>409 `location_mismatch`<br>422 Unprocessable | `02-accept-parcel` | `05-slot-unavailable.puml`, `06-booking-details.puml`, `08-empty-error.puml` |
| **US-2** (Перевірка слоту `storage`)<br>*Перегляд точного часу закінчення зберігання посилки.* | **DFD:** Читання з **D1**.<br>**ERD:** `parcels` (обчислюване поле `storage`). | `GET /api/Parcels/{id}` <br><br> `GetParcelByIdQuery` | **Time Rule:** Timeslot обчислюється як `arrived_at + 3 дні`. Усі дати в UTC. | 200 OK<br>404 `not_found`<br>401 Unauthorized | `03-view-delivery-point` | `02-availability.puml`, `06-booking-details.puml` |
| **US-3** (Скасування посилки)<br>*Миттєве вивільнення зайнятого місця на відділенні.* | **DFD:** Асинхронне оновлення **D1**.<br>**ERD:** `parcels`, `outbox_messages`. | `PATCH /api/Parcels/{id}/cancel` | **Еволюція статусів:** Перехід дозволено лише з валідних станів. | 200 OK<br>409 `wrong_parcel_status` | `02-accept-parcel` (керування) | `06-booking-details.puml`, `07-cancel-modal.puml` |
| **US-4** (Створення відділення)<br>*Налаштування нової точки з певним лімітом `capacity`.* | **DFD:** Оновлення **D2**.<br>**ERD:** `delivery_points`. | `POST /api/DeliveryPoint` <br><br> `CreateDeliveryPointCommand` | **Constraints:** Обов'язкове встановлення ліміту `capacity` для нової точки. | 201 Created<br>422 Unprocessable | `03-view-delivery-point` (опосередковано) | `01-home.puml` |
| **US-5** (Створення посилки)<br>*Реєстрація нової посилки в системі відправником.* | **DFD:** Процес **P2**.<br>**ERD:** `parcels`, `outbox_messages`. | `POST /api/Parcels` <br><br> `CreateParcelCommand` | **PII Masking:** Персональні дані клієнта (email, телефон, ім'я) маскуються в логах. | 202 Accepted<br>422 Unprocessable<br>401 Unauthorized | `01-create-parcel` | `03-booking-form.puml`, `04-booking-form-errors.puml`, `06-booking-details.puml` |
